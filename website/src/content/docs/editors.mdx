# 编辑器集成

LspProxy 通过替换 LSP 服务器的启动命令来工作，格式为：

```
lsp-proxy [lsp-proxy 选项] -- <原 LSP 命令> [LSP 参数...]
```

以下是各主流编辑器的配置方法。

import { Tabs, Callout } from 'nextra/components'

## Neovim

### 使用 nvim-lspconfig

在任意 LSP 的 `setup()` 调用中添加 `cmd` 字段：

```lua
local lspconfig = require('lspconfig')

-- rust-analyzer
lspconfig.rust_analyzer.setup({
  cmd = { 'lsp-proxy', '--', 'rust-analyzer' },
})

-- gopls（Go）
lspconfig.gopls.setup({
  cmd = { 'lsp-proxy', '--', 'gopls', 'serve' },
})

-- clangd（C/C++）
lspconfig.clangd.setup({
  cmd = { 'lsp-proxy', '--', 'clangd', '--background-index' },
})

-- pyright（Python）
lspconfig.pyright.setup({
  cmd = { 'lsp-proxy', '--', 'pyright-langserver', '--stdio' },
})

-- typescript-language-server（TypeScript/JavaScript）
lspconfig.ts_ls.setup({
  cmd = { 'lsp-proxy', '--', 'typescript-language-server', '--stdio' },
})
```

### 指定配置文件

```lua
lspconfig.rust_analyzer.setup({
  cmd = {
    'lsp-proxy',
    '--config', vim.fn.expand('~/.config/lsp-proxy/config.yaml'),
    '--',
    'rust-analyzer',
  },
})
```

### 临时切换引擎

```lua
lspconfig.gopls.setup({
  cmd = { 'lsp-proxy', '-e', 'openai', '--', 'gopls' },
})
```

### 使用 mason.nvim 管理 LSP

如果使用 mason-lspconfig，可通过 `handlers` 统一包装所有 LSP：

```lua
require('mason-lspconfig').setup_handlers({
  function(server_name)
    local config = require('lspconfig')[server_name]
    local default_cmd = config.document_config.default_config.cmd or { server_name }

    require('lspconfig')[server_name].setup({
      cmd = vim.list_extend({ 'lsp-proxy', '--' }, default_cmd),
    })
  end,
})
```

## VSCode

VSCode 各语言扩展的 LSP 服务器路径配置方式不同，以下是常见扩展的配置。

### rust-analyzer

在 `settings.json` 中：

```json
{
  "rust-analyzer.server.path": "/usr/local/bin/lsp-proxy",
  "rust-analyzer.server.extraEnv": {}
}
```

<Callout type="warning">
  VSCode 扩展通常直接调用 LSP 二进制，需要将 `lsp-proxy` 设置为服务器路径。但由于 `lsp-proxy` 需要 `--` 和 LSP 命令作为参数，部分扩展可能不支持直接传参。此时建议创建一个包装脚本。
</Callout>

**创建包装脚本（推荐）**

`~/.local/bin/rust-analyzer-proxy`：

```bash
#!/bin/sh
exec lsp-proxy -- rust-analyzer "$@"
```

```bash
chmod +x ~/.local/bin/rust-analyzer-proxy
```

然后在 `settings.json` 中：

```json
{
  "rust-analyzer.server.path": "/home/yourname/.local/bin/rust-analyzer-proxy"
}
```

### clangd（C/C++）

`~/.local/bin/clangd-proxy`：

```bash
#!/bin/sh
exec lsp-proxy -- clangd "$@"
```

`settings.json`：

```json
{
  "clangd.path": "/home/yourname/.local/bin/clangd-proxy"
}
```

## Zed

在 `~/.config/zed/settings.json` 中配置语言服务器的二进制路径和参数：

```json
{
  "lsp": {
    "rust-analyzer": {
      "binary": {
        "path": "lsp-proxy",
        "arguments": ["--", "rust-analyzer"]
      }
    },
    "gopls": {
      "binary": {
        "path": "lsp-proxy",
        "arguments": ["--", "gopls", "serve"]
      }
    },
    "clangd": {
      "binary": {
        "path": "lsp-proxy",
        "arguments": ["--", "clangd", "--background-index"]
      }
    }
  }
}
```

## Emacs

### 使用 lsp-mode

```elisp
(setq lsp-rust-analyzer-server-command '("lsp-proxy" "--" "rust-analyzer"))
(setq lsp-go-server-path "lsp-proxy")
(setq lsp-go-gopls-server-args '("--" "gopls" "serve"))
```

### 使用 eglot

```elisp
(add-to-list 'eglot-server-programs
             '(rust-mode . ("lsp-proxy" "--" "rust-analyzer")))

(add-to-list 'eglot-server-programs
             '(go-mode . ("lsp-proxy" "--" "gopls" "serve")))
```

## Helix

在 `~/.config/helix/languages.toml` 中覆盖语言服务器命令：

```toml
[[language]]
name = "rust"
language-servers = ["rust-analyzer-proxy"]

[[language]]
name = "go"
language-servers = ["gopls-proxy"]

[language-server.rust-analyzer-proxy]
command = "lsp-proxy"
args = ["--", "rust-analyzer"]

[language-server.gopls-proxy]
command = "lsp-proxy"
args = ["--", "gopls", "serve"]
```

## 通用包装脚本

对于不支持直接设置 LSP 参数的编辑器，可为每个 LSP 服务器创建包装脚本。

**模板：** `~/.local/bin/<lsp-name>-zh`

```bash
#!/bin/sh
# lsp-proxy 包装脚本：将 <lsp-name> 的文档翻译为中文
exec lsp-proxy -- <lsp-name> "$@"
```

**常用 LSP 包装示例：**

```bash
# rust-analyzer
echo '#!/bin/sh\nexec lsp-proxy -- rust-analyzer "$@"' > ~/.local/bin/rust-analyzer-zh
chmod +x ~/.local/bin/rust-analyzer-zh

# gopls
echo '#!/bin/sh\nexec lsp-proxy -- gopls "$@"' > ~/.local/bin/gopls-zh
chmod +x ~/.local/bin/gopls-zh

# clangd
echo '#!/bin/sh\nexec lsp-proxy -- clangd "$@"' > ~/.local/bin/clangd-zh
chmod +x ~/.local/bin/clangd-zh
```

之后在编辑器中将 LSP 服务器路径指向这些包装脚本即可。
